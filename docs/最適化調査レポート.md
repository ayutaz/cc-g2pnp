## 2. データパイプライン — GPU 利用率 10-15% (最重大)

### 2.1 ボトルネック: pyopenjtalk PnP ラベル生成

- `generate_pnp_labels()` が **~5-15ms/サンプル**
- バッチサイズ ~410 サンプル、4 ワーカー → ~1000ms/バッチ
- GPU forward+backward: ~100-150ms/バッチ
- **データ準備 / GPU 時間比 ≈ 7-10x → GPU は常時データ待ち**

### 2.2 推奨対策

| 優先度 | 施策 | GPU 利用率 |
|--------|------|-----------|
| 現状 | num_workers=4, リアルタイム PnP 生成 | ~10-15% |
| P0 | `persistent_workers=True` (1 行) | +1-3% |
| P0 | `num_workers=8` (1 行) | ~20-30% |
| **P0** | **PnP ラベル事前キャッシュ (LMDB/WebDataset)** | **~70-80%** |
| P1 | `prefetch_count=8-16` | ~80-90% |

## 3. Optimizer / Scheduler

### 3.1 `fused=True` — 最重要 1 行変更

```python
# optimizer.py:52
torch.optim.AdamW(param_groups, lr=..., betas=..., fused=True)
```

- T4 実測: optimizer.step() **24.66ms → 15.37ms (37.7% 削減)**
- 1.2M ステップで **~5 時間削減**
- T4 (CC 7.5) で完全サポート済み

### 3.2 8-bit Optimizer (bitsandbytes)

- AdamW state: 640 MB → 160 MB (**480 MB 削減**)
- 要 `uv add bitsandbytes`
- T4 15 GB で余裕がある場合は不要

## 7. torch.compile

### 7.1 重要な互換性問題

**`torch.compile + DDP + gradient_checkpointing` の 3 者同時使用は PyTorch 2.x で重大バグあり。**

- Issue #144035: `CheckpointError: different number of tensors saved`
- Issue #118984: `crashes with DDP + dynamic shapes + checkpoint`

### 7.2 推奨適用戦略

| 場面 | 適用 | 理由 |
|------|------|------|
| **推論** | **適用推奨** (`reduce-overhead`) | +30-50% 速度改善, リスク低 |
| 訓練 (GC 無効) | 検討可 | メモリ増加トレードオフ |
| 訓練 (GC + DDP) | **回避推奨** | 重大バグあり |
